GridWorld.h / GridWorld.cpp 管理二维网格环境，包括网格尺寸、障碍物分布 记录每个格子的访问状态和访问次数 提供坐标有效性检查、障碍物判断、覆盖率计算等功能 内部使用[y][x]索引优化内存访问，对外提供(x, y)接口保持直观

Walker.h / Walker.cpp 表示单个探索小车，记录当前位置和起始位置 提供移动功能，与GridWorld交互检查移动合法性 支持重置到起始位置（用于多次实验）

Config.h 配置结构体 集中管理所有模拟参数： 网格参数（宽、高、格子大小） 障碍物模式（无、随机密度、从文件加载） 小车数量、目标覆盖率、最大步数 运动规则（随机、有偏、避障） 碰撞处理规则（忽略、停止、反弹） 随机种子、实验次数等

MonteCarloSimulator.h 蒙特卡洛模拟引擎 协调GridWorld和Walker完成模拟流程 实现不同的运动规则算法： RandomMove：完全随机移动 BiasedMove：带方向偏好的移动 AvoidObstaclesMove：主动避开障碍物 处理碰撞逻辑 跟踪模拟状态（当前步数、覆盖率） 判断模拟终止条件（达到目标覆盖率或最大步数）

ExperimentRunner.h 批量实验管理器 运行多次独立实验以获取统计结果 支持参数扫描（改变某个参数观察影响） 计算均值、标准差等统计指标 管理实验结果数据集

SimulationResult.h 结果数据结构 定义单次模拟的结果格式（步数、最终覆盖率） 定义统计结果结构体（均值、标准差等） 工具类文件

Renderer.h / Renderer.cpp（待做） 图形渲染器 提供可视化界面，实时显示网格、障碍物、小车位置 处理用户输入（暂停、调整速度、重置等） 仅在交互模式下使用

CLI.h / CLI.cpp（代做） 命令行界面 提供简单的文本交互界面 实时输出模拟进度和状态

FileIO.h / FileIO.cpp (选做) 文件输入输出 从文件加载障碍物地图 导出实验结果到CSV文件 保存/加载配置文件 入口文件

main.cpp 程序入口 解析命令行参数 创建配置对象 根据运行模式（交互/批处理）选择执行路径 协调整个程序的启动和结束
